DataBuddy — E-Commerce Analytics Architecture

Mini Project — Artificial Intelligence Course · Streamlit + Ollama + Supabase

📦 Tech Stack

  • • Frontend: Streamlit (Python) — 4 pages
  • • Backend: Python — analytics, ML, data modules
  • • Reverse Proxy: Caddy 2 — Auto HTTPS, Let's Encrypt
  • • Database: Supabase (PostgreSQL) — 8 tables
  • • Infra: Tencent Cloud VPS — 2 CPU / 7.4GB RAM
  • • Orchestration: Docker Compose — 3 services
  • • LLM API: ZhipuAI GLM-4-Plus

🤖 ML Features

  • • Market Basket Analysis: Apriori algorithm — product affinity
  • • Customer Clustering: Segmentation via RFM + K-Means
  • • Sales Forecasting: Time series prediction (ARIMA/SARIMA)
  • • AI Chatbot: Context-aware via ZhipuAI GLM-4-Plus
  • • Local Models: Ollama — qwen2.5:1.5b & qwen2.5:3b
  • • Dashboard: Interactive charts, KPIs, business metrics

🚀 Deployment

  • • VPS: Tencent Cloud — 2 vCPU / 7.4GB RAM / 79GB disk
  • • Docker Compose: web (Streamlit) + ollama + caddy
  • • Security: HTTPS via Let's Encrypt, Caddy auto-renewal
  • • Volumes: ollama_data, caddy_data, app_data
  • • Domains: databuddy.my.id, ssh.databuddy.my.id
  • • ETL Pipeline: Load → Transform → Upload → Validate
  • • Data: 8,142 transactions across 8 tables
